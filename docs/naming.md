# Project naming

The project is named **Morpheus** (chosen 2026-05-15). The other candidates considered during Phase 0 are preserved below — useful if a subsystem within Morpheus wants a related name, or if a rename is ever revisited.

## Morpheus (chosen)

MOR-fee-us, 3 syllables. The form-giver in Greek myth; in *The Matrix*, the one who reveals reality. Fits a graph that shapes raw episodes into structured knowledge and surfaces the *why* behind any belief. Module reads naturally: `from morpheus import explain`. Group_id is lowercase `morpheus`.

## Argos

AR-gos, 2 syllables. The hundred-eyed watcher of Greek myth; ever-attentive. Direct metaphor for validators watching every reference across the graph. Original-feeling, low collision risk. Good candidate name for an internal validator service inside Morpheus.

## Janus

JAY-nus, 2 syllables. The two-faced Roman god of beginnings and transitions — looks both backward and forward. Fits the backtracking-on-reasoning core (every fact knows where it came from AND where it leads). Some collision with Janus session servers in the wider ecosystem.

## Mnemo

NEH-mo, 2 syllables. Short for Mnemosyne (Greek goddess of memory, mother of the Muses). Memory-flavoured, original, two syllables. Reads naturally as a Python package.

## Considered and rejected

- **Palimpsest** — beautiful metaphor (manuscript scraped clean and rewritten while the original remains faintly visible — exactly the append-only with tag-pointer revisions architecture) but too hard to pronounce.
- `justif`, `semant`, `prov-graph` — generic, insufficient differentiation, or look like typos.
- `vouch`, `deriv`, `because`, `belabor` — PyPI or GitHub collisions.
- `retract` — clean and TMS-shaped (names the central operation) but the user wanted something more catchy and AI-reasoning-flavoured.
